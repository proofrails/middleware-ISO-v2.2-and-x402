# API: x402

## Pricing

```
GET /v1/x402/pricing
```

Returns a list of protected endpoints with their prices. No auth required.

```json
[
  {
    "path": "/v1/x402/premium/verify-bundle",
    "price": "0.001",
    "currency": "USDC",
    "recipient": "0x..."
  }
]
```

## Premium endpoints

All premium endpoints require a valid `X-PAYMENT` header or return `HTTP 402`.

### X-PAYMENT header format

```json
{
  "tx_hash": "0x<transaction_hash>",
  "amount": "0.001",
  "recipient": "0x<recipient_address>",
  "currency": "USDC",
  "chain": "base"
}
```

Pass as the `X-PAYMENT` header value (JSON string).

### Verify bundle (premium)
```
POST /v1/x402/premium/verify-bundle
```
Body: `{"bundle_hash": "0x..."}`

### Generate statement (premium)
```
POST /v1/x402/premium/generate-statement
```
Body: `{"date": "2026-01-01"}`

### FX lookup (premium)
```
POST /v1/x402/premium/fx-lookup
```
Body: `{"base_ccy": "USD", "quote_ccy": "FLR", "provider": "ftso"}`

Providers: `ftso` (live FTSO v2), `fallback` (static rate).

### Bulk verify (premium)
```
POST /v1/x402/premium/bulk-verify
```
Body: `{"bundle_urls": ["https://..."]}`

### Refund via x402 (premium)
```
POST /v1/x402/premium/refund
```
Body: `{"original_receipt_id": "<uuid>", "reason_code": "CUST"}`

## 402 response body

```json
{
  "error": "payment_required",
  "message": "This endpoint requires a micropayment",
  "accepted": [
    {
      "amount": "0.001",
      "currency": "USDC",
      "recipient": "0x...",
      "chain": "base"
    }
  ]
}
```

Headers: `x-payment-required: true`

## Analytics

```
GET /v1/x402/payments?limit=50     # auth required
GET /v1/x402/revenue?days=7        # admin only
```

## Development mode

Set `X402_MOCK_PAYMENTS=true` to bypass on-chain verification. **Never use in production.**
